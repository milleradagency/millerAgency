This Support page was created to provide a better organizational workflow for website updates, changes, and bug fixes.

All updates to the website must be facilitated through this system. All emails to individuals about website changes will be disregarded.

This ticket system allows content writers, admins, and web developers the ability to organize, reply, and track any and all general issues, changes, feature requests, and bugs.

Please use this system reponsiblity. It helps the entire agency manage this website and present ourselves professionally.

For additional information, read through the other items in the FAQs (Frequently Asked Questions). Thank you.

To submit a new ticket to the system, click the "Create a Ticket" link on the Support page. You'll be presented with a form, similar to a standard contact form, where you can fill out the necessary relevant information.

The following list defines what each form field means in the order that they appear.

SUBJECT*
The subject field. Similar to an email, write out a short summary of your ticket's contents. Subjects will be displayed on the "Tickets" list. Subject lines should be short and descriptive. This field is required.
DESCRIPTION*
A detailed account of the relevant aspects, characteristics, or information of the subject you wish to present. Please fill out the ticket description with as much information as possible to fully convey the purpose of your submission. This field is required.
CATEGORY*
Each ticket is placed in a category to help us sort issues. Please see the, "What do the categories mean?" FAQ section for more information. This field is required.
PRIORITY*
Each new ticket in the system is given a priority level which designates its urgency.. Please see the, "What do the priority levels mean?" FAQ section for more information. This field is required.
ATTACHMENTS
Upload and attach any relevant screenshots, documents (Word, PDF), etc that pertain to your ticket submission.
SUBMIT TICKET
Click the submit ticket button to place your submission into the system.
RESET FORM
Clears all data from the form and resets it back to it's default state.

Each ticket is placed in a category to help us sort issues.

BUG
A bug is an error, flaw, failure, or fault in the website that causes it to produce an incorrect or unexpected result, or to behave in unintended ways.
CHANGE REQUEST
A change request is a call for an adjustment of the website; all change requests will be reviewed before determining its implementation.
FEATURE REQUEST
Feature requests are ideas. They propose enhancements and additions to the website. All feature requests will be extensively reviewed before determining its usefulness and possible implementation.
GENERAL
Tickets that do not fit in any other category will be filed here.

Each ticket is a assigned a status which designates it's current relevance.

OPEN
OPEN tickets are issues that have yet to be resolved.
ON HOLD
ON HOLD tickets are issues that have been reviewed and concluded to be, for now, unresolvable.
CLOSED
CLOSED tickets are issues that have been resolved.

Each new ticket in the system is given a priority level which designates its urgency.

LOW
These issues are cosmetic and do not afflict the performance, visibility, or usability of the website.
NORMAL
Normal issues may cause minor interference with the website, but not with enough severity to be considered Medium or High priority. Chances are these issues will not affect a large user base.
MEDIUM
The next step up from Normal priority; Medium issues need to be addressed soon, but are not critically urgent.
HIGH
All issues at this priority must be fixed as soon as possible.
